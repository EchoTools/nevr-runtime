================================================================================
TECHNICAL NOTES
================================================================================

Hook Implementation:
  - Uses MinHook library for function hooking
  - Preserves original function pointers for call forwarding
  - Uses fastcall calling convention (__fastcall)
  - 64-bit addressing (uint64_t, uintptr_t)
  - Safe base address verification before hook installation
  - MH_Initialize checks for already-initialized state

Logging:
  - Uses fprintf + fflush for real-time output
  - GetTickCount() for millisecond timestamps
  - Hex formatting for pointers (0x%llx)
  - Signed decimal for state values (%lld)
  - No buffering to ensure immediate visibility

Integration:
  - Initializes in deferred thread (avoids loader lock)
  - Cleanup in DLL_PROCESS_DETACH
  - try/catch around all initialization/cleanup
  - Diagnostic logging to dllmain_deferred_init.txt

Build System:
  - CMake integration with MinGW cross-compile
  - Static linking for standalone deployment
  - vcpkg dependency management
  - Automatic dist target packaging

================================================================================
TROUBLESHOOTING
================================================================================

If hooks don't install:
  - Check base address readiness in log
  - Verify game version (target: May 3, 2023 build)
  - Check RVA addresses match binary version

If no function calls logged:
  - Verify weapon is equipped
  - Check if weapon actually fires (visual feedback)
  - Test with booster hook first (simpler baseline)

If game crashes:
  - Function signature may be incorrect
  - Check calling convention (should be __fastcall)
  - Verify parameter types match Ghidra analysis

Performance:
  - Minimal overhead (~50-100 µs per function call)
  - No impact on gameplay at typical fire rates
  - Log file size: 1-5 KB per weapon fire event
